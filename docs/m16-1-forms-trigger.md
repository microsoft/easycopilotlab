# 실습: Forms 트리거 연결하기
{: .no_toc }

| 시간 | 소요 | 수강생 역할 |
|:-----|:-----|:-----------|
| 17:25 | 20분 | 🟢 직접 실습 |

---

## Step 1 — Forms 설문 준비

Microsoft Forms에서 **HR 문의 폼**을 만듭니다.

1. [Microsoft Forms](https://forms.office.com) 접속 → **+ 새 양식**
2. 제목: `HR 문의`
3. 질문 추가:

| # | 질문 | 유형 |
|:--|:-----|:-----|
| 1 | 이름 | 텍스트 |
| 2 | 부서 | 텍스트 |
| 3 | 문의 내용 | 텍스트(긴 답변) |

{: .note }
> 폼을 만든 뒤 **공유** 버튼으로 링크를 복사해두면 테스트할 때 편리합니다.

---

## Step 2 — 에이전트 트리거 추가

HR 도우미 에이전트에 **"Forms에 새 응답이 제출되면"** 이벤트 트리거를 연결합니다.

1. Copilot Studio → **HR 도우미** 에이전트
2. 상단 탭 **"+6" (더보기)** 또는 **"트리거"** 메뉴 클릭
3. **"+ 트리거 추가"** → **"Forms에 새 응답이 제출되면"** 선택
4. 위에서 만든 **HR 문의** 폼 선택 → **저장**

{: .tip }
> 트리거를 추가하면 에이전트가 **사용자의 대화 없이도** Forms 제출 이벤트로 자동 실행됩니다.

---

## Step 3 — 지침에 트리거 동작 규칙 추가

**지침** 섹션의 `## STRICT RULES` 끝에 아래 내용을 **추가**하세요:

```
- Forms 트리거로 에이전트가 시작되면:
  - 문의 내용을 분석하여 답변 초안을 작성
  - 담당자 메일(hr@abc.co.kr)로 아래 내용을 전달:
    · 제목: "[HR 문의] {이름} - {부서}"
    · 본문: 원문 문의 내용 + AI 답변 초안
  - flow_HRRequest 흐름을 활용하여 메일 발송
```

에이전트가 트리거로 깨어나면, 지침을 보고 **문의 내용 분석 → 답변 초안 생성 → 담당자에게 메일 전달**을 자동으로 수행합니다.

---

## Step 4 — 테스트

1. Forms에서 테스트 문의 제출:
   - 이름: 홍길동
   - 부서: 마케팅팀
   - 문의 내용: "복지포인트 잔여 확인 방법을 알고 싶습니다"
2. 에이전트가 트리거로 자동 실행되는지 확인
3. 담당자 메일(hr@abc.co.kr)에 **원문 문의 + AI 답변 초안**이 도착하는지 확인

{: .warning }
> 테스트 후 반드시 **게시(Publish)**하세요. 게시하지 않으면 트리거가 실제 환경에서 작동하지 않습니다.
