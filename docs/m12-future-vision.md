## 에이전트 하나에서 팀으로

한 에이전트가 HR도, IT도, 경비도 모두 담당하기는 어렵습니다.  
사람도 한 명이 모든 부서 업무를 하기 어렵듯이요.

해결책은 **에이전트 팀**을 만드는 것입니다.

## 멀티에이전트 오케스트레이션

Copilot Studio에서는 **에이전트를 다른 에이전트의 도구처럼 연결**할 수 있습니다.

### 작동 방식

```mermaid
graph TD
    A[👤 사용자 질문] --> B[🎯 중앙 에이전트]
    B -->|HR 관련| C[🏢 HR 에이전트<br>도구로 연결]
    B -->|IT 관련| D[💻 IT 에이전트<br>도구로 연결]
    B -->|경비 관련| E[💰 경비처리 에이전트<br>도구로 연결]
    C --> F[💬 답변]
    D --> F
    E --> F
```

사용자는 **하나의 에이전트**와 대화하지만, 뒤에서는 **다른 에이전트들이 도구로 호출**됩니다.

### Copilot Studio에서의 설정

1. 전문 에이전트(HR, IT, 경비)를 **각각 만들고 게시**
2. 중앙 에이전트의 **"도구" 설정**에서 다른 에이전트를 추가
3. 중앙 에이전트의 지침에 "HR 관련 질문은 HR 에이전트를 호출" 등 규칙 작성
4. 중앙 에이전트가 **질문을 판단하고, 적합한 에이전트를 자동 호출**

| 항목 | 설명 |
|:-----|:-----|
| **중앙 에이전트** | 사용자와 대화하며 요청을 판단·분배 |
| **전문 에이전트** | 도구로 연결되어 특정 분야 처리 후 결과 반환 |
| **연결 방식** | Copilot Studio "도구" 메뉴에서 에이전트 추가 |

### 확장 상상

전문 에이전트를 하나씩 추가로 만들어서 중앙 에이전트의 도구로 연결하면,  
회사 전체를 커버하는 **에이전트 조직**이 됩니다.

{: .highlight }
> 각 전문 에이전트를 만드는 방법은 **오늘 배운 것과 동일**합니다. 그 에이전트를 도구로 연결하기만 하면 됩니다.

---

## 트리거 — 자율 에이전트

지금까지 에이전트는 사용자가 **말을 걸어야** 답했습니다.  
트리거를 붙이면 에이전트가 **스스로 움직입니다.**

| 종류 | 설명 | 비유 | 예시 |
|:-----|:-----|:-----|:-----|
| **일정 기반** | 정해진 시간에 자동 실행 | 매일 아침 출근하는 직원 | 매일 오전 9시 뉴스 브리핑 |
| **이벤트 기반** | 특정 사건 발생 시 실행 | 전화 오면 받는 안내원 | 새 메일 도착 시 자동 분류 |
| **에이전트 간** | 다른 에이전트가 호출 | 부서 간 업무 요청 | 멀티에이전트 연계 |

### 일정 트리거 예시

```
매일 오전 9시 자동 실행
    ↓
에이전트가 뉴스 브리핑 생성
    ↓
Teams에 자동 게시
    ↓
여러분이 출근하기 전에 이미 정보 수집 완료
```

{: .tip }
> 트리거가 연결되면 에이전트는 **스스로 일을 시작**합니다. 여러분이 출근도 하기 전에요.

## MCP — 에이전트의 USB 포트

### MCP란?

**MCP(Model Context Protocol)**는 에이전트가 외부 도구·데이터에 연결하는 **표준 규격**입니다.

USB가 어떤 기기든 꽂으면 바로 쓸 수 있듯이,  
MCP를 지원하는 도구는 **어떤 에이전트든 꽂으면 바로 연결**됩니다.

| 비유 | 설명 |
|:-----|:-----|
| **USB 이전** | 프린터마다 다른 케이블, 마우스마다 다른 포트 |
| **USB 이후** | 하나의 포트로 모든 기기 연결 |
| **MCP 이전** | 도구마다 다른 연결 방식 (커스텀 커넥터, API별 설정) |
| **MCP 이후** | 하나의 표준으로 모든 도구·데이터 연결 |

### Copilot Studio + MCP

Copilot Studio는 **MCP 서버 연결을 지원**합니다.

```mermaid
graph LR
    A[🤖 에이전트] -->|MCP| B[🗄️ 사내 DB]
    A -->|MCP| C[📊 SAP/ERP]
    A -->|MCP| D[🔍 사내 검색]
    A -->|MCP| E[🌐 외부 API]
    A -->|MCP| F[📁 파일 시스템]
```

설정 방법:
1. Copilot Studio → 에이전트 → **"도구"** 메뉴
2. **"MCP 도구 추가"** 선택
3. MCP 서버 URL 입력
4. 연결 완료 → 에이전트가 해당 도구를 자동으로 사용

### 왜 중요한가?

| Before (커스텀 연동) | After (MCP) |
|:---------------------|:------------|
| 도구마다 커넥터를 따로 만들어야 함 | MCP 서버 URL만 입력하면 끝 |
| 개발자가 API 연동 코드 작성 | 문과생도 URL 하나로 연결 |
| 도구가 바뀌면 다시 개발 | MCP 표준이라 교체가 쉬움 |

{: .highlight }
> MCP는 에이전트 세계의 USB입니다. **한 번 표준이 정해지면, 모든 것이 쉽게 연결됩니다.**

## 실전 시나리오: Forms 문의 → AI 답변 초안 → 관리자 검토

실제 고객사에서 요청이 들어온 시나리오입니다.  
오늘 배운 기술이 어떻게 연결되는지 살펴봅시다.

### 시나리오

> "회사 내부에 문의 폼을 만들어 게시하고, 직원이 문의를 접수하면 AI가 **답변 초안**을 만들어서 관리자에게 보내주세요.  
> 관리자는 초안을 검토한 후 **직접 최종 대응**합니다."

### 전체 흐름

```mermaid
flowchart LR
    A[📋 직원이<br>Forms 작성] --> B[⚡ Power Automate<br>이벤트 트리거]
    B --> C[🤖 AI 프롬프트<br>답변 초안 생성]
    C --> D[📧 관리자에게<br>초안 메일 전달]
    D --> E[👤 관리자<br>검토 + 최종 대응]
```

### 어떤 기술이 필요한가?

| 단계 | 기술 | 오늘 배운 모듈 |
|:-----|:-----|:-------------|
| ① Forms 접수 시 자동 시작 | 이벤트 트리거 (Forms 응답) | 이 모듈 (M12 트리거) |
| ② 문의 내용으로 답변 초안 생성 | AI 프롬프트 — 텍스트 유형 | M11 |
| ③ 관리자에게 초안 메일 전달 | Flow + Office 365 메일 발송 | M9 |
| ④ 관리자가 검토 후 최종 대응 | **Human-in-the-Loop** 패턴 | 설계 원칙 |

### Human-in-the-Loop — AI 시대의 핵심 설계 원칙

AI가 아무리 잘해도, **최종 결정은 사람**이 합니다.

| 패턴 | 설명 | 위험도 |
|:-----|:-----|:------|
| **완전 자동** | AI 답변 → 직원에게 곧바로 전송 | ⚠️ 오답 시 신뢰 하락 |
| **Human-in-the-Loop** ✅ | AI 초안 → 관리자 검토 → 승인 후 전송 | ✅ 품질 보장 |

{: .important }
> AI는 **초안을 만드는** 역할, 사람은 **검토하고 최종 결정**하는 역할.  
> 이것이 가장 안전하고 효과적인 AI 협업 패턴입니다.

### 이 시나리오를 만들려면?

1. **Microsoft Forms** — 문의 양식 만들기 (제목, 카테고리, 상세 내용)
2. **Power Automate** — Forms 응답 도착 시 자동 트리거
3. **AI 프롬프트 (텍스트)** — 문의 내용 + 사내 FAQ 기반으로 답변 초안 생성
4. **메일 발송** — 관리자에게 원문 + 답변 초안을 함께 메일 전달
5. **관리자** — 초안을 검토·수정 후 직접 답변

오늘 배운 **M9(Flow + 메일) + M11(AI 프롬프트) + M12(트리거)**를 조합하면  
**내일 당장 만들 수 있는** 시나리오입니다.

---

## 당장 멀티에이전트를 만들어야 하나요?

{: .note }
> **아닙니다!** 오늘 만든 에이전트 하나를 잘 운영하는 게 우선입니다. 필요할 때 확장하면 됩니다.

---

## 핵심 정리

1. **멀티에이전트** = 전문 에이전트를 도구로 연결하여 팀으로 운영
2. **MCP** = 에이전트의 USB 포트 — 표준 규격으로 어떤 도구든 쉽게 연결
3. **트리거 3종** = 에이전트가 스스로 움직이는 방법 (일정/이벤트/에이전트 간)
4. **Human-in-the-Loop** = AI 초안 + 사람 검토 — 가장 안전한 AI 협업 패턴
5. 먼저 에이전트 하나를 잘 운영하고, **필요할 때 확장**

{: .highlight }
> "에이전트 하나의 완성에서 에이전트 팀으로 — 트리거가 연결되면 스스로 움직이는 디지털 동료가 됩니다."

## FAQ

| 질문 | 답변 |
|:-----|:-----|
| 멀티에이전트 설정이 어렵나요? | Copilot Studio의 "도구" 메뉴에서 다른 에이전트를 추가하면 됩니다. 클릭 몇 번으로 연결됩니다. |
| 트리거가 오작동하면? | Power Automate 실행 기록에서 확인하고 조건을 세밀하게 설정할 수 있습니다. |
| 멀티에이전트는 추가 비용이 있나요? | 에이전트 수에 따라 라이선스가 달라질 수 있습니다. 관리자와 확인하세요. |
| 에이전트끼리 서로 충돌하지 않나요? | 중앙 에이전트의 지침에 각 전문 에이전트의 역할을 명확히 쓰면 충돌 없이 협업합니다. |
