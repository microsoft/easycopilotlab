# 실습: 지침 업그레이드 + 테스트
{: .no_toc }

| 시간 | 소요 | 수강생 역할 |
|:-----|:-----|:-----------|
| 13:00 | 15분 | 🟢 직접 작성 + 테스트 |

---

## 지침 템플릿

M3에서 에이전트 빌더에 입력한 지침은 간단했습니다.  
이제 Copilot Studio에서 **더 정교한 지침으로 업그레이드**하세요.

Copilot Studio → HR 도우미 에이전트 → **지침** 섹션의 내용을 아래로 교체합니다:

<details markdown="1">
<summary><strong>지침 (클릭해서 펼치기)</strong></summary>

```
## 역할
당신은 우리 회사의 HR/총무 전담 도우미입니다.

## 범위
복리후생, 연차/휴가, 경비처리, 사내 규정에 관한 질문에만 답변합니다.

## 태도
- 한국어 존칭을 사용합니다
- 핵심을 먼저 말하고, 부가 설명은 뒤에
- 200자 이내로 간결하게

## 원칙
- 지식에 없는 내용: "정확한 답변을 드리기 어렵습니다. 
  HR팀(내선 1234)에 문의해 주세요"
- 개인정보(급여, 인사평가): 담당자 연결 안내
```

</details>

{: .note }
> 아직 지식을 연결하지 않았기 때문에 회사 고유 규정은 정확히 답하지 못할 수 있습니다. 이 모듈에서는 **범위**, **말투**, **민감정보 처리 원칙**이 잘 작동하는지만 확인합니다. 정확한 회사 규정 답변은 다음 모듈(M7)에서 검증합니다.

## 테스트 3종

지침을 입력한 후, 아래 3가지 질문으로 테스트하세요:

| # | 질문 | 기대 반응 |
|:--|:-----|:---------|
| 1 | "연차 며칠이야?" | ✅ 범위 안 — 일반 답변 또는 보수적 안내 |
| 2 | "오늘 날씨 어때?" | 🚫 범위 밖 — 답변 범위가 아님을 안내 |
| 3 | "내 급여가 얼마야?" | 🔒 원칙 적용 — 담당자 연결 안내 |

{: .tip }
> 지침에서 "존칭"을 "반말"로 바꾸고 다시 테스트해 보세요. 텍스트 한 줄의 변경이 에이전트를 완전히 바꿉니다.
