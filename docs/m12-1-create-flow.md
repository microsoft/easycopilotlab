## 완성 흐름 구조

| 항목 | 내용 |
|:-----|:-----|
| **흐름 이름** | flow_HRRequest |
| **트리거** | 에이전트가 흐름을 호출할 때 |
| **입력 ①** | `myRequest` (텍스트): 문의 내용 |
| **입력 ②** | `mySender` (텍스트): 문의자 이름 |
| **입력 ③** | `myEmail` (텍스트): 담당자 메일 주소 |
| **동작 ①** | AI 프롬프트 실행 — 메일 본문 HTML 생성 |
| **동작 ②** | Office 365 Outlook — 메일 보내기(V2) |
| **출력** | `myReturn`: 완료 메시지 |

---

## Step 1 — 새 에이전트 흐름 만들기

1. Copilot Studio 좌측 메뉴에서 **"흐름"** 클릭
2. **"+ 새 에이전트 흐름"** 클릭

   ![흐름 목록 — 새 에이전트 흐름](../assets/images/m12/image.png)

---

## Step 2 — 트리거 설정

1. 트리거 추가 화면에서 **"에이전트"** 검색
2. **"에이전트가 흐름을 호출할 때"** 선택

   ![트리거 검색 — 에이전트가 흐름을 호출할 때](../assets/images/m12/image2.png)

---

## Step 3 — 입력 변수 3개 추가

1. 트리거 카드에서 **"+ 입력 추가"** 클릭

   ![입력 추가 버튼](../assets/images/m12/image3.png)

2. 사용자 입력 유형에서 **"텍스트"** 선택

   ![사용자 입력 유형 — 텍스트 선택](../assets/images/m12/image4.png)

3. 같은 방법으로 **3개 입력 변수**를 추가:

   | 입력 이름 | 유형 | 설명 |
   |:---------|:-----|:-----|
   | `myRequest` | 텍스트 | 문의 내용 |
   | `mySender` | 텍스트 | 문의자 이름 |
   | `myEmail` | 텍스트 | 담당자 메일 주소 |

   ![입력 변수 3개 완성](../assets/images/m12/image5.png)

---

## Step 4 — 출력 노드 설정

트리거 아래에 자동으로 추가되는 **"에이전트에게 응답"** 카드를 먼저 설정합니다.

1. 하단의 **⊕** 버튼 → 작업 추가 → **"에이전트에게 응답"** 선택

   ![작업 추가 — 에이전트에게 응답](../assets/images/m12/image6.png)

2. **"+ 출력 추가"** 클릭

   ![출력 추가 버튼](../assets/images/m12/image7.png)

3. 텍스트 유형으로 출력 변수 `myReturn` 추가

   ![출력 변수 myReturn 추가](../assets/images/m12/image8.png)

---

## Step 5 — AI 프롬프트 추가 (메일 본문 생성)

트리거와 응답 사이에 AI 프롬프트를 넣습니다.

1. 트리거와 응답 **사이의 ⊕ 버튼** 클릭

   ![트리거와 응답 사이 + 버튼](../assets/images/m12/image9.png)

2. 작업 추가 → AI 기능 → **"프롬프트 실행"** 선택

   ![작업 추가 — 프롬프트 실행](../assets/images/m12/image10.png)

3. 프롬프트 드롭다운 → **"+ 새 사용자 지정 프롬프트"** 클릭

   ![새 사용자 지정 프롬프트](../assets/images/m12/image11.png)

---

### 5-1. AI 프롬프트 작성

1. 프롬프트 이름: **myRequestMail**
2. 하단 **"Copilot 시작하기"** 입력란에 다음을 입력하고 전송:

   ```
   내가 요청내용을 입력하면 그걸 받아 메일 본문용 HTML 코드로 변환
   ```

   ![프롬프트 편집기 — Copilot으로 지침 생성](../assets/images/m12/image12.png)

3. Copilot이 생성한 지침을 확인 → **"유지"** 클릭

   ![Copilot이 생성한 프롬프트 지침 — 유지 클릭](../assets/images/m12/image13.png)

{: .note }
> Copilot이 자동 생성한 프롬프트 지침에는 HTML 변환 규칙(p, br, strong 태그 사용 등)이 포함됩니다. 필요에 따라 수정할 수 있습니다.

---

### 5-2. 프롬프트 입력 변수 추가

1. **"+ 콘텐츠 추가"** 또는 하단 **"→ 1개 입력"** 클릭
2. 텍스트 입력 추가:
   - **이름**: `Request Text`
   - **샘플 데이터**: 테스트용 문구 입력 (예: "외국인 직원을 위한 영어버전, 중국어 버전의 회사 규정 문서도 제공해 주세요")

   ![텍스트 입력 — Request Text 추가](../assets/images/m12/image14.png)

---

### 5-3. 프롬프트 테스트

1. 상단 **"테스트"** 버튼 클릭
2. **모델 응답** 영역에 HTML 코드가 생성되는 것 확인
3. 결과가 만족스러우면 **"저장"** 클릭

   ![프롬프트 테스트 — HTML 결과 확인 후 저장](../assets/images/m12/image15.png)

{: .important }
> AI Builder 프롬프트는 AI Builder 크레딧이 필요합니다. 조직 정책에 따라 사용 불가일 수 있습니다. 이 경우 AI 프롬프트 단계를 건너뛰고, 메일 본문에 `myRequest` 변수를 직접 넣으세요.

---

### 5-4. 프롬프트에 입력 변수 매핑

흐름 디자이너로 돌아오면 프롬프트 실행 카드가 보입니다.

1. **프롬프트**: `myRequestMail` 이 선택된 것 확인
2. **Request Text** 필드 클릭 → 우측 **번개(⚡)** 아이콘 클릭

   ![프롬프트 실행 — Request Text에 동적 콘텐츠 매핑](../assets/images/m12/image16.png)

3. 동적 콘텐츠 패널에서 **"에이전트가 흐름을 호출할 때"** → **`myRequest`** 선택

   ![동적 콘텐츠 — myRequest 선택](../assets/images/m12/image17.png)

4. Request Text에 `myRequest` 변수가 매핑된 것 확인

   ![Request Text에 myRequest 매핑 완료](../assets/images/m12/image18.png)

---

## Step 6 — 메일 보내기 동작 추가

1. 프롬프트 실행 아래 **⊕** 버튼 → 작업 추가 → **"outlook"** 검색
2. Office 365 Outlook → **"메일 보내기(V2)"** 선택

   ![작업 추가 — outlook 검색 → 메일 보내기(V2)](../assets/images/m12/image19.png)

3. 메일 보내기 카드가 추가됨 — 받는 사람, 제목, 본문 필드 확인

   ![메일 보내기(V2) 카드](../assets/images/m12/image20.png)

---

### 6-1. 받는 사람 — 동적 콘텐츠로 변경

1. **받는 사람** 필드 오른쪽 **톱니바퀴(⚙)** 클릭 → **"동적 콘텐츠 사용"** 선택

   ![받는 사람 — 동적 콘텐츠 사용 전환](../assets/images/m12/image21.png)

2. 번개(⚡) 아이콘 → 동적 콘텐츠에서 **`myEmail`** 선택

---

### 6-2. 제목·본문 입력

| 필드 | 입력 값 |
|:-----|:-------|
| **받는 사람** | `myEmail` (동적 콘텐츠) |
| **제목** | `요청 또는 문의사항 입니다` (직접 입력) |
| **본문** | 프롬프트 실행 결과 `Text` (동적 콘텐츠) |

1. **제목**: `요청 또는 문의사항 입니다` 직접 입력
2. **본문** 필드 클릭 → **번개(⚡)** 아이콘 클릭

   ![받는 사람 myEmail + 제목 + 본문 번개 아이콘](../assets/images/m12/image22.png)

3. 동적 콘텐츠 패널 → **프롬프트 실행** → **`Text`** 선택

   ![본문 — 프롬프트 실행 Text 선택](../assets/images/m12/image23.png)

4. 메일 보내기 카드 완성 확인

   ![메일 보내기 완성 — 본문에 프롬프트 실행 결과](../assets/images/m12/image24.png)

---

## Step 7 — 게시

1. 우측 상단 **"게시"** 클릭
2. "에이전트 흐름이 게시되었습니다" 메시지 확인

   ![게시 완료 메시지](../assets/images/m12/image25.png)

---

## Step 8 — 흐름 테스트

게시 후 흐름이 정상 작동하는지 테스트합니다.

1. 상단 **"테스트"** 클릭 → **"수동"** 선택 → **"테스트"** 클릭

   ![흐름 테스트 — 수동 선택](../assets/images/m12/image26.png)

2. 입력값 입력:

   | 입력 | 테스트 값 |
   |:-----|:---------|
   | **myRequest** | `안녕하세요 요청사항이 있어요. 1. 요청111 2. 요청 222 3. 요청 333` |
   | **mySender** | `삼돌이` |
   | **myEmail** | 본인 메일 주소 |

   → **"흐름 실행"** 클릭

   ![테스트 입력값 입력 → 흐름 실행](../assets/images/m12/image27.png)

3. **"흐름 실행이 시작되었습니다"** ✅ 확인

   ![흐름 실행 성공](../assets/images/m12/image28.png)

4. 입력한 메일 주소로 가서 **실제 메일 수신 확인** 🎉

   ![수신된 메일 확인](../assets/images/m12/image29.png)

---

## Step 9 — 흐름 이름 변경

1. 상단 **"개요"** 탭 → 세부 정보 옆 **"편집"** 클릭
2. **흐름 이름**: `flow_HRRequest` 로 변경
3. **"저장"** 클릭

   ![흐름 이름 변경 — flow_HRRequest](../assets/images/m12/image30.png)

{: .tip }
> 흐름 이름에 `flow_` 접두어를 붙이면 Copilot Studio에서 도구를 추가할 때 **쉽게 검색**할 수 있습니다.
